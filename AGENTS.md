## 1. Project Overview

LeelaClue is a **mindfulness and self-discovery mobile app** inspired by the ancient Indian game of Leela (Snakes & Arrows). This repository hosts the **marketing website** at `https://leelaclue.com`.

The website is a **static site served by GitHub Pages**. The landing pages (`index.html`) for all three languages are **generated by a Node.js build script** from Markdown source files. All other pages (blog, faq, user_guide, etc.) remain hand-crafted HTML.

**Current App Version**: v1.7.3

## 3. Critical Rules

### 3.1 Trilingual Consistency
- **Every content change to landing pages must be applied to all three language MD files** in `assets/docs/`.
- After editing MD files, always run `node build_html.js` to regenerate HTML.
- German uses informal **"du"** (lowercase) consistently.
- Russian uses informal **"СӮСӢ"** consistently, with correct **"С‘"**.

### 3.2 Landing Page Editing Workflow

**The `en/index.html`, `de/index.html`, and `ru/index.html` are generated files вҖ” never edit them directly.**

To change landing page text:
1. Edit the relevant `assets/docs/landing_[section]_[lang].md` file(s)
2. Run `node build_html.js` (or `npm run build`)
3. Commit both the MD source and the generated HTML

Sections and their MD files:
| Section | Text file | Carousel images directory |
|---|---|---|
| Hero / About | `landing_hero_[lang].md` | `assets/images/carousel/hero/` |
| SВ·OВ·R Framework | `landing_sor_[lang].md` | `assets/images/carousel/sor/` |
| Six Steps / Practice | `landing_practice_[lang].md` | `assets/images/carousel/practice/` |
| Case Study: Anna | `landing_anna_[lang].md` | `assets/images/carousel/anna/` |
| Daily Card / App | `landing_daily_[lang].md` | `assets/images/carousel/daily/` |

**To change carousel images:** drop `.webp` files into the corresponding directory and run `node build_html.js`. The build script reads all `.webp` files from the directory, sorted alphabetically. No code changes needed.

### 3.3 Build Script (`build_html.js`)

The build script:
- Reads 5 MD files per language вҶ’ converts with `marked` вҶ’ injects into the full HTML template
- Generates the left-panel section navigation with localized dot labels
- Generates `llms-full.txt` aggregating all EN content for AI discovery
- Outputs `en/index.html`, `de/index.html`, `ru/index.html`

When changing the build script, also update the **CSS version query string** (e.g. `?v=10`) to bust browser caches, then rebuild.

```bash
node build_html.js
```

### 3.4 Navigation Order
Navigation bar order: **Offline Game | Community | Guide** (left to right).
When updating navigation, update the `getTemplate()` function in `build_html.js` (for the landing pages) AND all hand-crafted HTML files across all 3 languages.

### 3.5 Shared Assets
- **One CSS file**: `assets/css/style.css` вҖ” all styling is here.
- **One main JS**: `assets/js/script.js` вҖ” carousels, scroll-spy section nav, SEO-safe scroll animations, hamburger menu, privacy banner, voting.
- **Font**: `Philosopher` (from `assets/fonts/`). Used for headings and brand elements.
- **Body font**: `Segoe UI, Tahoma, Geneva, Verdana, sans-serif`.

### 3.6 Design System
- **Colors**: Primary Gold `#d4af37` В· Background `#0a0a12` В· Secondary BG `#151520` В· Text `#e0e0e0`
- **Aesthetic**: Dark, premium, spiritual. Gold accents, glassmorphism, radial gradients.
- **No placeholder images** вҖ” use real assets only.

### 3.7 Sitemap Integrity вҖ” Line Endings
**`sitemap.xml` must always use LF-only (`\n`) line endings.** Mixed CRLF+LF caused GSC "couldn't fetch" errors for weeks (confirmed April 2026).

- Any script writing to `sitemap.xml` must normalize endings: `content.replace(/\r\n/g, '\n')`.
- Verify after any modification: `python -c "open('sitemap.xml','rb').read().count(b'\r')"` вҶ’ must be `0`.
- After fixing, remove the sitemap from GSC and resubmit.

### SEO-safe scroll animations
- **Before JS runs**: all content fully visible (`opacity: 1`) вҶ’ Google and LLMs read everything
- **After JS runs**: `body.js-anim` class is added вҶ’ `.animate-on-scroll` elements fade in on scroll
- This ensures text is always indexable, animations are progressive enhancement only

### Carousel responsive sizes
| Breakpoint | Width | Height |
|---|---|---|

| 1200px | 440px | 520px |
| 1100px | 380px | 460px |

Fixed-width approach (copied from relimie pattern) prevents carousels from stretching full-width and looking distorted on mobile.

## 5. SEO & AI Discoverability

### Implemented
- All landing page text is baked into static HTML at build time вҖ” no `opacity:0` blocking, no JS-dependent rendering for content
- `llms.txt` at site root вҖ” linked from every page `<head>` with `<link rel="alternate" type="text/plain" title="LLM Context">`
- `llms-full.txt` вҖ” aggregated EN landing content + user guide, regenerated on every build
- `SoftwareApplication` + `WebSite` JSON-LD on all landing pages
- Canonical URLs and hreflang on all pages
- LCP preload for hero carousel first image

### GSC Indexability
- Use **GSC вҶ’ URL Inspection вҶ’ View Rendered Page** to confirm all 5 sections render with visible text
- The build system ensures text is always in the DOM regardless of JS execution

## 6. Common Patterns

### Update landing page text (all languages)
```bash
# 1. Edit the MD files
# assets/docs/landing_hero_en.md, landing_hero_de.md, landing_hero_ru.md ...

# 2. Rebuild
node build_html.js

# 3. Commit
git add assets/docs/ en/index.html de/index.html ru/index.html llms-full.txt
git commit -m "content: update hero section text"
```

### Update announcement badges (version / blog post)
Edit the `leftPanel()` function in `build_html.js` вҖ” update `versionLabel`, `postTag`, and the `href` values. Then rebuild.

### Batch-editing hand-crafted HTML pages
```js
const fs = require('fs');
const path = require('path');
const root = 'c:/GitHub/leelaclue.github.io';
['en','de','ru'].forEach(lang => {
    const dir = path.join(root, lang);
    fs.readdirSync(dir)
      .filter(f => f.endsWith('.html') && f !== 'index.html')
      .forEach(file => {
        const fp = path.join(dir, file);
        let c = fs.readFileSync(fp, 'utf8');
        c = c.replace(/OLD_TEXT/g, 'NEW_TEXT');
        fs.writeFileSync(fp, c, 'utf8');
    });
});
```
**Note**: `index.html` is excluded above вҖ” it's a generated file, edit via `build_html.js` instead.
**Note**: PowerShell on this machine does not support `&&` chaining. Use `;` or Node.js scripts.

### Adding a new hand-crafted page
1. Create in `/en/`, `/de/`, `/ru/`
2. Include standard header (logo, nav, lang-switch, hamburger) and footer
3. Add to `sitemap.xml` with hreflang alternates
4. Update nav if needed (edit `getTemplate()` in `build_html.js` + all hand-crafted HTML files)

### Announcing a new blog post
1. Create content in `assets/posts/` per language
2. Update `assets/js/blog.js` and `assets/js/translations.js`
3. Run `node build_blogs.js` to generate static HTML blog pages
4. Update `leftPanel()` in `build_html.js`: change `postTag` hrefs and labels, then rebuild
5. Run `node update_sitemap.js` вҖ” ensure LF-only line endings (**critical**, see В§3.7)
6. Add pulsing dot `<span class="dot-new"></span>` to "Community" in nav across all files

### Bumping app version
1. Update `versionLabel` strings in `leftPanel()` in `build_html.js`
2. Update JSON-LD `softwareVersion` in `getSchemaOrg()` in `build_html.js`
3. Update version references in hand-crafted HTML files
4. Rebuild: `node build_html.js`
